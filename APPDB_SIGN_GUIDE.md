# AppDb.to 在线签名使用指南

使用 AppDb.to 免费在线签名，可以直接发送链接给女朋友，她用手机就能安装。

---

## 📋 准备工作

### 你需要准备的：
1. ✅ GitHub 账号（已有）
2. ✅ IPA 文件（用 GitHub Actions 构建）
3. ✅ AppDb.to 账号（免费注册）

---

## 🚀 操作步骤

### 第一步：构建 IPA

1. 访问你的 GitHub 仓库
2. 进入 **Actions** 页面
3. 点击 **Run workflow** 触发构建
4. 等待 5-10 分钟
5. 下载 `iOS-IPA-NoCodesign` 文件

### 第二步：注册 AppDb.to

1. 访问：https://appdb.to
2. 点击右上角 **Register** 注册
3. 填写信息完成注册
4. 验证邮箱

### 第三步：上传 IPA 并签名

1. 登录 AppDb.to
2. 点击顶部 **Apps** → **Store**
3. 点击 **+ Upload App** 按钮
4. 上传你的 IPA 文件
5. 填写应用信息：
   ```
   Name: 情侣待办清单
   Version: 1.0.0
   Description: 我和芬芬的专属待办清单~
   ```
6. 点击 **Submit** 提交
7. 等待处理（约 1-3 分钟）

### 第四步：获取安装链接

1. 处理完成后，进入 **My Apps**
2. 找到你上传的应用
3. 点击应用详情
4. 复制 **Install Link**（安装链接）

### 第五步：发送给女朋友

```
链接格式类似：
https://appdb.to/app/xxxxxxxx

直接把这个链接发给女朋友微信~
```

---

## 📱 女朋友如何安装

### 方法一：直接点击链接

1. 用 iPhone 打开你发的链接
2. 点击 **安装** 按钮
3. 按照提示操作：
   - 设置 → 通用 → VPN 与设备管理
   - 信任企业级应用
   - 点击信任

### 方法二：扫码安装

AppDb.to 提供 QR 码，可以直接扫码安装

当应用过期后：

4. 发新链接给女朋友

## 💡 小技巧

### 保存应用信息
首次上传时，AppDb.to 会保存你的应用信息，后续重新签名时：
- 只需上传新的 IPA 文件
- 链接保持不变
- 更方便快捷

### 多设备安装
同一个链接可以安装到多台设备：
- 你的 iPhone
- 女朋友的 iPhone
- 以后换手机也能用

---

## ⚠️ 注意事项

1. **网络问题**：AppDb.to 偶尔会不稳定，多试几次
2. **证书撤回**：免费证书可能被苹果撤回，需要重新签名
3. **描述文件**：安装前需要在设置中信任描述文件

---

## 📞 遇到问题？

### 常见问题解决

| 问题 | 解决方法 |
|------|---------|
| 无法安装 | 检查 iOS 版本，建议 iOS 13+ |
| 证书无效 | 重新上传 IPA 获取新证书 |
| 链接失效 | 重新生成安装链接 |
| 下载缓慢 | 切换网络或稍后重试 |
